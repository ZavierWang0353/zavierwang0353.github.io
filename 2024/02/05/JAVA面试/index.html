进程和线程

进程是程序运行的基本单位，一个进程就是一个执行中的程序，它在计算机中一个指令接着一个指令地执行着

线程是一个比进程更小的执行单位。一个进程在其执行的过程中可以产生多个线程。享同一块内存空间和一组系统资源。切换工作时，负担要比进程小得多

1）好处 使用多线程可以把程序中占据时间长的任务放到后台去处理

2）坏处 大量的线程降低代码的可读性。 更多的线程需要更多的内存空间。 当多个线程对同一个资源出现争夺时候要注意线程安全的问题。

临界区表示一种公共资源或者说是共享数据只能有一个线程使用它，一旦临界区资源被占用，其他线程要想使用这个资源，就必须等待

线程一共有五个状态，分别如下： **新建(new)：可运行(runnable等待被线程调度选中)：运行(running)

Thread类中包含的成员变量代表了线程的某些优先级。在默认情况下优先级都是Thread.NORM_PRIORITY（常数5）（max和min分别一到10）。

interrupted()方法的解释，测试当前线程是否已经中断。线程的中断状态由该方法清除。 换句话说，如果连续两次调用该方法，则第二次调用返回false。

isInterrupted()并未清除状态

stop()方法停止线程则是非常暴力的，会抛出java.lang.ThreadDeath异常，通常情况下，此异常不需要显示地捕捉。

将方法interrupt()与return结合使用也能实现停止线程的效果。

异步

如果两个线程同时操作对象中的实例变量在方法前加上synchronized关键字即可。

静态方法是属于类的而不属于对象的。synchronized修饰的静态方法锁定的是这个类的所有对象.

Synchronized还可作用于一个类

synchronized修饰方法和修饰一个代码块类似，只是作用范围不一样，修饰代码块是大括号括起来的范围，而修饰方法范围是整个函数。

synchronized关键字加在方法上还是对象上，如果它作用的对象是非静态的，则它取得的锁是对象；如果synchronized作用的对象是一个静态方法或一个类，则它取得的锁是对类，

锁的重入性，即在同一锁程中，线程不需要再次获取同一把锁。

cas乐观锁，默认不会冲突，出现冲突则返回旧值。所以出现冲突时参与的线程只有一个会成功

比如一个旧值A变为了成B，然后再变成A，刚好在做CAS时检查发现旧值并没有变化依然为A：添加一个版本号可以解决。

使用CAS时非阻塞同步，自旋就是一个死循环。如果JVM能支持处理器提供的pause指令，那么在效率上会有一定的提升。

只能保证一个共享变量的原子操作 。有一个解决方案是利用对象整合多个共享变量

volatile

Lock前缀的指令会引起处理器缓存写回内存； 一个处理器的缓存回写到内存会导致其他处理器的缓存失效； 当处理器发现本地缓存失效后，就会从内存中重读该变量数据，即可以获取当前最新值。

每个线程对应一个工作内存，并共享主内存的数据，

对于普通变量：读操作会优先读取工作内存的数据，如果工作内存中不存在，则从主内存中拷贝一份数据到工作内存中；写操作只会修改工作内存的副本数据

volatile变量，读操作时JMM会把工作内存中对应的值设为无效，要求线程从主内存中读取数据；写操作时JMM会把工作内存中对应的数据刷新到主内存中

保证各个CPU的缓存一致性，每个CPU通过嗅探在总线上传播的数据来检查自己缓存的数据有效性
