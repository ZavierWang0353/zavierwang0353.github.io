springboot笔记

发表于2024-02-05|更新于2024-02-15|Work
|阅读量:

Springboot 和 Spring的区别。Spring算是Springboot的前身，改进的地方主要是在配置方面的自动化。

Spring Boot的配置依旧沿用maven（也有用gradle的）

根目录下的application.property记录了一些基本的配置比如url和端口这一类

spring boot的参数配置来源多样且有优先级:

  • 命令行
  • java系统属性
  • 操作系统环境变量
  • application

定位视图

Spring IoC

管理JAVA Bean的容器(java bean可视作一种可以被序列化的类)

IoC可通过名称和类别来寻找类, 特别的分为两种方式原型(prototype)和单例(singleton)

@Bean在引入第三方库的时候比较好用, 一般都会有内置的构造函数

@Autowired可用在类的内部, 会自动注入依赖类,但是autowired不能处理歧义接口

  1. 在类实现接口的时候注释**@Primary**, 但多个类声明primary的话依旧不能解决冲突问题
  2. 在autowired注解@Qualified(“Classname”)

Bean的生命周期

  1. 扫描时的资源定位
  2. bean的定义(不在IoC)
  3. IoC装在Bean定义
  4. 创建Bean对象
  5. 依赖注入

在Bean注入时, 可在类中重写相关函数,在实例化/初始化/摧毁的时候都可以重载函数来进行处理

属性文件默认从 application.property中获取,相关的依赖包名: spring-boot-configuration-processor

Bean的作用域可以在注解后声明,默认为prototype. 还有singleton, session和application. 主要是看在哪段区域内返回同一个bean(如prototype永远返回不一样的bean)

Profile可以用于调试. 在Bean注解下写注明生效的profile. 同时如果在根目录新建application-{profile.name}.properties, 也会沿用新的设置

在启动项目时可标注profile

xml也可用来定义bean

Spring AOP

约定

给核心函数打包，包括前置工作和后续处理，也会针对函数运行结果执行不同的代码（try except final）

ProxyBean

创建需要三个参数，类加载器（实际逻辑），接口（定义after，before，return这些），逻辑实现（after，before，return内部顺序的实现，这部分springboot源码承担）

AOP

用途：动态环境配置，自动err处理

常用属于：

  • join point，需要被包装的方法
  • point cut， 分辨众多join point的正则式
  • advice，before，after等函数的定义
  • target， 一般为join point的父对象
  • introduction，用@DeclareParents注入后可通过强制类型转换来调用新增的功能。
  • aspect，可能会存在多个切面，可用order来定义优先级，优先级下呈嵌套关系。

DataBase

在maven中添加相关依赖，在application.property中添加以下依赖：

类型 作用
url 数据库地址
user 用户名
password 密码
driver 可选，springboot会尽量自动选driver
max_idle 最大等待连接
max-active 最大活跃
max-wait 最大等待
initial-size 初始化连接量

jdbcTemplate每次在有新的请求进入时会重新建立一次到数据库的连接。

jdbcTemplate.query(带参数的sql语句,sql语句中的参数, 对应数据库类型的构建映射）

JPA不用像jdbctemplate一样手写类型，而是在类的定义通过使用注解来和类联系起来。

@EnableJpaRepositories和@EntityScan指定JPA接口和JPA实体类的内容

同时JPA的语言支持（JPQL）和Hibernate（HQL）是很类似的。非常灵活。也包括模糊查询。

MyBatis更倾向于用xml，但也可以使用注解

首先需要一个数据类，其中针对特殊的类型比如enum需要定义一个typehandler的实体类

在typerhandler<自定义数据类名>前定义@MappedJDBCType(jdbc中的数据类型)和@MappedType(java类型)来实现转换，主要的方法有getNullableResult和setNonNullbaleParameter.

之后在xml文件中链接sql语句和对应的类中的方法(类中可为虚函数),类前表明@respository来完成自动注入.
